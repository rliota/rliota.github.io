Projects

  • JSCompile

    A tool to facilitate good JavaScript code organization practices in Java shops. (Getting a Java team to support Node.js in their build process to leverage Browserify is tough work.)

    Similar to Browserify, it helps package code modules (files) with dependency statements to other modules into a consolidated single JavaScript file. It differs from Browserify’s interface in that it recursively traverses a set of files under a specified root project directory, rather than dynamically by way of an entry file. I've been thinking of adding support for relative path dependency statements to the project, but haven’t had the time to do it yet.
