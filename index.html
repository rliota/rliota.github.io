Projects

  •  JSCompile

    (2014, Java)

    JSCompile is a tool for pulling together multiple related JavaScript files into one single resource file. It traverses through a given directory path, building namespace objects based on the sub-directories it finds.

    For example, this directory structure:

    • main/
      • util/
      • view/
        • Library.js
      • model/
        • Book.js
        • Word.js
      • controller/
        • LibraryManager.js
    might produce a file that would look something like: 
        function initialize_MyJSResources(){
        Main = {};
        Main.util = {};
        Main.view = {};
        Main.model = {};
        Main.controller = {};
        Main.model.Word = function(){
            ...
        }

        Main.model.Book = function(){
            ...
        }

        Main.view.Library = function(){
            ...
        }

        Main.controller.LibraryManager = function(){
            ...
        }
    }

    After the namespaces are defined, the compiler begins writing out the .js resources starting from the deepest directory and working its way back up to the root. As it does this, it strips out comments and also keys into any @import annotations defined there to determine object definition dependencies. If one file's object depends on another file's, simply placing an import annotation in the comments of the dependent file will prompt the compiler to ensure that the required file is written to the output file before the working one is. (i.e. if main/model/Book.js had the annotation @import Main.model.Word, main/model/Word.js would be pulled in before Book.js.)

    JSCompile keeps track of which files it has already traversed, so any @imported file is guaranteed to only be loaded once. Presently, the compiler does not parse the javascript within the files however, and has no knowledge of what objects exist within them, so it cannot resolve/detect incorrect object definitions. This means that you could define an object like Not.a.namespace.Reader in a file under "util" called "Sunshine.js", and the compiler would not instantiate the mis-matched namespace.
